# Task Workflow

## Standard Workflow

```
1. Read requirements (PRD → SRS → SDD → procedures → phase)
2. Understand the scope of the current phase
3. Implement one task at a time within the phase
4. Verify implementation against requirements
5. Create Manual Testing Checklist
6. Stop and present results to user
7. Wait for user approval before next phase
```

## Phase Execution Steps

1. **Preparation** — Read all relevant documents.
2. **Implementation** — Write code following coding standards and architecture.
3. **Review** — Check against the phase's Expected Output.
4. **Checklist** — Produce the Manual Testing Checklist.
5. **Handover** — Report summary of changes and wait for user.

## Bug Fixing Workflow

```
1. User reports bug from manual testing
2. Read the bug description carefully
3. Identify the issue in the current phase code
4. Fix only the reported bug (no new features)
5. Update Manual Testing Checklist with retest items
6. Present changes to user
```

## Feature Addition Workflow

```
1. User requests new feature
2. Analyze impact on PRD, SRS, SDD, procedures, and phases
3. Propose documentation changes if needed
4. Wait for explicit user approval
5. Only then begin implementation
```
