# AI Behavior Guidelines

## Core Principles

1. **Follow Phase Discipline** — Only work on the current active phase. Never skip ahead.
2. **Single Source of Truth** — PRD, SRS (SKS), and SDD are the primary references.
3. **No Self-Validation** — Never claim features are working, tested, or bug-free.
4. **Stop at Phase End** — Always stop after completing a phase. Wait for user approval.
5. **No Scope Creep** — Never add features outside documented requirements without explicit user permission.

## Prohibited Actions

- Writing backend code, APIs, databases, or external services.
- Creating new image/video assets without user instruction.
- Modifying PRD, SRS, or SDD without explicit user request.
- Generating placeholder images or downloading assets from the internet.
- Assuming testing results or build success.

## Required Behavior

- Read all relevant documents before starting any phase.
- Provide a Manual Testing Checklist at the end of each phase.
- Summarize changes and files modified after each phase.
- Wait for user instructions before proceeding to the next phase.
